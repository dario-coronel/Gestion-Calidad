# Glosario de Términos

## Términos Generales

### Auditoría
Revisión sistemática de procesos, productos o sistemas para verificar el cumplimiento de estándares. Puede ser interna (personal de la empresa) o externa (terceros certificadores).

### Conformidad
Cumplimiento de requisitos, especificaciones o estándares establecidos.

### No Conformidad (NC)
Incumplimiento de un requisito o estándar de calidad. Debe ser reportada, investigada y resuelta.

### Calidad
Grado en que un producto o servicio cumple con las expectativas y requisitos del cliente y la organización.

## Términos de Gestión

### Norma
Estándar o requisito de calidad que la organización se compromete a cumplir. Agrupa puntos relacionados.

**Ejemplo:** ISO 9001:2015, BPM (Buenas Prácticas de Manufactura)

### Punto de Norma
Requisito específico dentro de una norma. Es el nivel más granular de especificación.

**Ejemplo:** Punto 8.1 de ISO 9001 - "Planificación operacional"

### Causa Raíz
Razón fundamental por la cual ocurrió una no conformidad. Identificarla permite diseñar soluciones permanentes.

**Ejemplos:**
- Falta de capacitación
- Proceso no documentado
- Falla de equipamiento

### Orden de Mejora (OM)
Acción correctiva o preventiva derivada de una NC o iniciativa de mejora. Define qué se debe hacer, quién y cuándo.

### Acción Correctiva
Medida tomada para eliminar la causa de una no conformidad.

### Acción Preventiva
Medida tomada para evitar que ocurra una no conformidad potencial.

## Términos Operacionales

### Hallazgo
Resultado de una verificación/auditoría. Puede ser conforme (OK) o no conforme (problema).

### Verificación
Auditoría o inspección para validar el cumplimiento de estándares.

### Lote
Conjunto de productos fabricados bajo las mismas condiciones. Se identifica y rastrea con QR.

### Trazabilidad
Capacidad de rastrear un producto desde su origen hasta su destino final. El sistema QR facilita esto.

### Sector
Área o departamento de la organización (Producción, Calidad, Logística, etc.).

## Términos de Proyecto

### Proyecto
Iniciativa temporal de mejora continua con objetivos definidos, responsable y plazo.

### Subtarea
Desglose de un proyecto en actividades específicas más pequeñas.

### Hito
Evento importante dentro de un proyecto (inicio, fase completada, cierre, etc.).

### Entregable
Resultado concreto que debe producir un proyecto.

## Estados y Estados

### Estados de NC

| Estado | Significado |
|--------|------------|
| **Abierta** | Recién reportada, pendiente análisis |
| **En Análisis** | Se investiga causa raíz |
| **En Cierre** | Solución implementada, en validación |
| **Cerrada** | Resuelta y verificada |

### Estados de OM

| Estado | Significado |
|--------|------------|
| **Por Ejecutar** | Asignada pero no iniciada |
| **En Ejecución** | Trabajo en progreso |
| **Por Validar** | Completada, pendiente aprobación |
| **Validada** | Aprobada, cierre confirmado |

### Estados de Proyecto

| Estado | Significado |
|--------|------------|
| **Planificación** | En definición |
| **En Ejecución** | Activo, trabajo en progreso |
| **Suspendido** | Pausado temporalmente |
| **Cerrado** | Completado |
| **Cancelado** | Abandonado |

## Términos de Clasificación

### Categoría de Hallazgo

| Categoría | Definición | Urgencia |
|-----------|-----------|----------|
| **No Conforme Mayor** | Incumplimiento grave de requisito | Alta |
| **No Conforme Menor** | Incumplimiento leve | Media |
| **Observación** | No es incumplimiento pero mejora es recomendada | Baja |

### Prioridad

| Nivel | Rango | Plazo Típico |
|-------|-------|-------------|
| **Alta** | 1 | Inmediato (< 1 semana) |
| **Media** | 2 | Corto plazo (1-4 semanas) |
| **Baja** | 3 | Largo plazo (> 1 mes) |

## Definiciones Normativas

### Requisito
Especificación o condición que debe cumplirse. Puede ser:
- Técnico (especificación del producto)
- Administrativo (documento, registro)
- Operacional (cómo se realiza el proceso)

### Especificación
Documento detallado que describe características, medidas, tolerancias y requisitos de un producto o proceso.

### Registro
Documento que evidencia que algo se realizó (firma, fecha, datos).

### Procedimiento
Documento que describe paso a paso cómo se realiza una actividad.

### Evidencia
Prueba documental o física que demuestra conformidad o no conformidad.
